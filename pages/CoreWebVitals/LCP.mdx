# Largest Contentful Paint (LCP)

Quick check for [Largest Contentful Paint](https://web.dev/articles/lcp), a Core Web Vital that measures loading performance. LCP marks when the largest content element becomes visible in the viewport.

**LCP Rating Thresholds:**

| Rating | Time | Meaning |
|--------|------|---------|
| 🟢 Good | ≤ 2.5s | Fast, content appears quickly |
| 🟡 Needs Improvement | ≤ 4s | Moderate delay |
| 🔴 Poor | > 4s | Slow, users may abandon |

> **Need to optimize?** Use [LCP Subparts](/CoreWebVitals/LCP-Subparts) to identify which phase (TTFB, load delay, load time, render delay) is causing the bottleneck.

### Understanding LCP

**What can be an LCP element:**
- `<img>` elements
- `<image>` inside `<svg>`
- `<video>` poster images
- Elements with `background-image` (CSS)
- Block-level text elements (`<p>`, `<h1>`, etc.)

**LCP updates until:**
- User interacts (click, scroll, keypress)
- A larger element renders
- Page load completes

### Common Causes of Slow LCP

| Cause | Solution |
|-------|----------|
| Slow server response | Optimize TTFB, use CDN |
| Render-blocking resources | Defer non-critical CSS/JS |
| Slow resource load | Preload LCP image, optimize size |
| Client-side rendering | Use SSR or prerender |
